# Korean Vocab — 延世韩国语单词记忆

基于间隔重复（Spaced Repetition）的韩语单词记忆 Web App，覆盖《延世韩国语》第 1-10 章共 877 个单词。

## 功能概览

- **分章学习** — 10 个章节卡片列表，点击进入学习
- **闪卡记忆** — 正面韩语、反面中文释义，点击/空格翻转
- **FSRS 间隔重复** — 基于记忆稳定性的自适应调度算法，每个单词有独立的复习间隔
- **交叉学习** — 复习词与新词按 2:1 比例交替排列，提升长期记忆效果
- **自适应新词** — 待复习多时自动减少新词，避免记忆负担过重
- **困难词系统** — 自动识别反复遗忘的单词，提供专门加强练习
- **今日复习** — 首页横幅自动统计今日到期单词，一键批量复习
- **进度统计** — 未学 / 学习中 / 复习中 / 已掌握 / 困难 五档实时统计 + 进度条
- **学习历史** — 按日期、章节汇总学习记录，各章掌握率一目了然
- **云同步** — 通过 GitHub Personal Access Token 将 `progress.json` 同步到远程仓库，多设备共用
- **导入 / 导出** — JSON 文件备份与恢复学习进度
- **键盘快捷键** — `空格` 翻牌、`←` 不认识、`→` 认识

## 核心逻辑

### 1. 单词数据

所有单词以内联 JSON 的形式直接写在 `index.html` 的 `CHAPTERS` 数组中，每章包含 `chapter`、`count`、`words`（`{ korean, chinese }`）三个字段。

### 2. 学习流程

```
首页(章节列表) → 点击章节 → 组装队列(复习+新词交叉) → 翻牌 → 判断认识/不认识 → 下一张 → 本章结束
```

每次进入章节时，`buildSessionQueue()` 组装临时队列：

1. **到期复习词** — 该章中 `nextReview <= 当前时间` 且未掌握、非困难的词
2. **新词** — 按顺序推进（每章维护 `newIndex` 指针），取下一批新词
3. **自适应数量** — 待复习 > 30 时新词减半，> 60 时只推 1 个，> 100 时不推新词
4. **交叉排列** — 复习词和新词按 2:1 比例交替，而非先复习后新词

### 3. 间隔重复算法（简化版 FSRS）

采用基于**记忆稳定性（Stability）**的调度模型，灵感来自 Anki 的 FSRS 算法。

每个单词维护以下状态：

| 字段 | 说明 |
|------|------|
| `status` | `learning` / `reviewing` / `mastered` / `leech` |
| `stability` | 记忆稳定性（天数），值越大表示记忆越持久 |
| `difficulty` | 单词难度 [0,1]，难度高的词稳定性增长慢 |
| `reps` | 累计答对次数（永不归零） |
| `lapses` | 累计答错次数（永不归零） |
| `streak` | 当前连续答对次数（答错归零） |
| `nextReview` | 下次复习的时间戳 |
| `lastReview` | 最后一次复习的时间戳 |

**核心常量：**

| 常量 | 值 | 说明 |
|------|------|------|
| `S0_FIRST_OK` | 0.4 天（~10h） | 新词首次答对的初始稳定性 |
| `S0_AGAIN` | 1/(24*60) 天（~1min） | 答错时的稳定性 |
| `STABILITY_GROWTH` | 1.35 | 每次答对稳定性乘数 |
| `STABILITY_SHRINK` | 0.25 | 答错时稳定性降至的比例 |
| `TARGET_RETENTION` | 0.9 | 目标记忆保持率 90% |
| `MASTERED_INTERVAL_DAYS` | 21 | 掌握阈值（复习间隔天数） |

**答对（认识）：**
- 首次答对：stability = 0.4 天，difficulty = 0.3
- 后续答对：difficulty 轻降，stability 增长 `S * 1.35 * (1 - 0.4*D)`
- 下次复习间隔 = `stability * 0.105`（由 R=0.9 的遗忘曲线推导）
- 间隔 >= 21天 → `mastered`

**答错（不认识）：**
- stability 降至原来的 25%（不归零，保留部分学习成果）
- difficulty 升高，streak 归零，lapses +1
- 1 分钟后再次复习，追加到本次队列末尾

**间隔增长示例（难度 0.3 的普通词）：**

| 复习次数 | stability | 下次间隔 |
|:---:|:---:|:---:|
| 1 | 0.4天 | ~1小时 |
| 4 | 0.98天 | ~1天 |
| 8 | 3.28天 | ~3.5天 |
| 14 | 19.9天 | ~21天 → mastered |

### 4. 困难词（Leech）系统

- 累计答错 >= 5 次 → 标记为 `leech`，从正常队列中排除
- 首页显示橙色横幅，点击进入专门的困难词复习 session
- 困难词连续答对 3 次 → 恢复为 `learning`，lapses 减 2

### 5. 数据存储

```
localStorage
├── korean_vocab_progress   — 每个单词的学习状态（JSON 对象，key 为韩语原文）
├── korean_vocab_history    — 每次 session 的记录（数组）
├── korean_vocab_daily      — 每日新词数量设置
└── korean_vocab_sync_token — GitHub PAT
```

### 6. 云同步机制

- 配置 GitHub PAT 后，每次修改进度会延迟 3 秒触发 `syncToCloud`
- 上传时通过 GitHub Contents API 更新仓库中的 `progress.json`，使用 `sha` 实现乐观并发控制
- 下载时以 `lastReview` 时间戳更大的版本为准，保留最新进度
- 历史记录取长度更长的版本
